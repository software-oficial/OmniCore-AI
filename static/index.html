Documentación Técnica

La API de OmniCore-AI es Command-Driven. No llamas a funciones, ejecutas comandos.

El Gateway Principal

Todos los comandos de negocio pasan por un único punto de entrada.

POST /api/gateway/execute

Envía un comando y recibe una respuesta estandarizada.

Header: Authorization: Bearer LEARNING_abc123... Body: { "command": "stock.add", "params": { "name": "iPhone 15", "price": 999.99, "quantity": 5 } }

📚 Catálogo de Comandos

Consulta el /api/agent/manifest para ver todos los comandos disponibles.

  • stock.add: Agrega productos al inventario.
  • stock.list: Lista todos los productos disponibles.
  • sales.process: Registra una nueva venta.
  • whatsapp.process: Gestiona la lógica de un mensaje entrante.

Blueprint para Frontend

Si estás construyendo una interfaz para OmniCore-AI, sigue este plan arquitectónico:

1. Gestión de Estado de Autenticación

Almacena el token obtenido en el registro en localStorage o sessionStorage.

const token = localStorage.getItem('omnicore_token');

2. Patrón de Comunicación (Command Pattern)

Crea un servicio centralizado de API que envuelva todas las llamadas al Gateway.

async function executeCommand(command, params = {}) { const response = await fetch('/api/gateway/execute', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${token}` }, body: JSON.stringify({ command, params }) }); return response.json(); }

3. Manejo de Respuestas (ServiceResponse)

El sistema siempre responde con el mismo formato, facilitando el manejo de errores:

{ "success": true/false, "data": { ... }, "message": "Descripción del resultado", "error_code": "CODE_XYZ", "latency_ms": 12.4 }
